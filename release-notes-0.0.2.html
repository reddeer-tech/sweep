What's New

Provider API key testing & smart model dropdown

  • Test button next to each API key in OpenClaw settings (OpenAI, Anthropic, Google, OpenRouter). Click Test and the app actually hits the provider's /models endpoint to verify the key works — the status pill turns green for valid, red for invalid (with a tooltip showing the provider's error), or yellow while testing.
  • Save automatically tests every non-empty key, so you get instant validation when you paste in a fresh key.
  • Default Model dropdown is now populated from real fetched models, not a hardcoded list. Only models from providers whose key tested as valid show up — because the available models genuinely differ from key to key based on your account tier, region, and beta access.
  • Tested provider models also flow up to the web and mobile chat clients, so you see the same dropdown everywhere.

Reliability fixes

  • Fixed: two devices on the same account would collide. Two Macs under one account with the same default hostname (the common <user>-MacBook-Pro.local) used to silently share a single machine row and machine token, making them appear as one device to the relay. Each device now gets its own independent identity, online status, and chat traffic.
  • Fixed: machine recovery now refreshes your login first. If your machine token gets invalidated (e.g. you removed the device from another client), the app refreshes your auth token before re-registering — so you stay logged in instead of getting bounced to the login screen.
  • Better release notes formatting. The Sparkle update dialog now renders headings, lists, links, code, blockquotes, and emphasis properly.
